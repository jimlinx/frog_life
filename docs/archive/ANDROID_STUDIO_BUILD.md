# Build and Install Using Android Studio

## The gradlew issue
The Gradle wrapper jar file is missing. This needs to be generated by Android Studio.

## ✅ Complete Steps to Build and Install

### Step 1: Open Project in Android Studio

1. **Launch Android Studio**
2. **File** → **Open**
3. Navigate to: `/Users/jimlin/Repos/frog_life`
4. Click **Open**

### Step 2: Sync Gradle Files (IMPORTANT!)

When the project opens:
1. You'll see a notification: **"Gradle files have changed"**
2. Click **"Sync Now"**
3. **Wait for sync to complete** (bottom status bar shows progress)
   - This downloads Gradle wrapper
   - Downloads all dependencies (AppCompat, Compose, etc.)
   - May take 2-5 minutes first time

### Step 3: Clean Project

1. **Build** menu → **Clean Project**
2. Wait for completion

### Step 4: Rebuild Project

1. **Build** menu → **Rebuild Project**
2. Wait for build to complete
3. Check "Build" tab at bottom - should say "BUILD SUCCESSFUL"

### Step 5: Uninstall Old Version

**On your device/emulator:**
- Go to **Settings** → **Apps** → **Frog Life** → **Uninstall**

**Or if you have ADB:**
```bash
adb uninstall com.froglife
```

### Step 6: Run the App

1. Click the green **Run** button (▶️) at top
2. Or: **Run** menu → **Run 'app'**
3. Select your device/emulator
4. Wait for installation
5. App should launch automatically

### Step 7: Check Launcher

After installation:
1. Go to your device home screen
2. Open app drawer
3. Look for **"Frog Life"** with green frog icon 🐸
4. Tap to open

## If App Still Doesn't Show in Launcher

### Fix 1: Clear Launcher Cache
1. **Settings** → **Apps**
2. Find your **Launcher app** (might be called "Launcher3", "Pixel Launcher", etc.)
3. **Storage** → **Clear Cache**
4. Go back to home screen

### Fix 2: Restart Launcher
1. **Settings** → **Apps** → **Default apps** → **Home app**
2. Select a different launcher
3. Go back and select your original launcher again

### Fix 3: Restart Device
Simply restart your device - this clears all launcher caches

### Fix 4: Check Installation
In Android Studio, look at the **Run** tab at bottom:
- Should say: "App installed successfully"
- Should say: "Launching 'MainActivity'"

If you see errors, read the error message carefully.

## Common Build Errors and Solutions

### Error: "Sync failed"
**Solution**:
- Check internet connection
- **File** → **Invalidate Caches** → **Invalidate and Restart**
- Try sync again

### Error: "SDK not found"
**Solution**:
- **Tools** → **SDK Manager**
- Install Android SDK Platform 34
- Install Android SDK Build-Tools 34

### Error: "Build failed"
**Solution**:
1. Read the error in Build tab
2. **Build** → **Clean Project**
3. **Build** → **Rebuild Project**

### Error: "App not installed"
**Solution**:
1. Check device is connected (green device icon in toolbar)
2. Enable USB debugging on device
3. Allow USB debugging popup on device
4. Uninstall old version manually

If you prefer command line, generate the wrapper first:

```bash

# This creates gradle-wrapper.jar
# Then you can use ./gradlew commands
```
